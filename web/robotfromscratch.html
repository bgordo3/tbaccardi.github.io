For anyone who wants to build their own RC Arduino robot, there are a few things you'll need: 1.The first component, obviously, is an Arduino. The better the Arduino, the more options you will have for your robot. 2.The next component you will need is an RC transmitter/receiver combination. For my robot, I used a Turnigy 9x 9 channel transmitter because I had it lying around from a previous project. A transmitter like this is complete overkill for a differential drive robot. The highest number of channels that you'll need for a simple RC platform is 3 (one for the throttle vector, one for steering, and potentially one for a reverse switch.) However, the number of channels you purchase for use with your robot should depend largely on what model Arduino you buy. An Arduino Uno only has 2 interrupt channels (see interrupts section). Interrupts are vastly superior than polling for signals that necessitate an immediate response, especially so when your CPU speed is limited (such as an Arduino). Therefore, you will probably want to only buy a 2-channel transmitter if you are using an Arduino Uno, unless you plan on upgrading to a Mega or Beaglebone Black in the future. 3.Next up is the frame. The frame should roughly depend on what you want the platform to do. I 3D printed my own frame at school to imitate the RedBot from Sparkfun (note: it seems as though the popular redbot chassis has been replaced by the Shadow Chassis. I can't speak to the quality of the newer chassis, the basic style is the same). I seem to have lost my Creo Parametric CAD file that has the chassis drawn out, but if I manage to find it I will upload it to my GitHub repo.

Robot from Scratch - Beaglebone

The Robot From Scratch project originated from a desire to create a more advanced design for a robot build for a class project. The class project was based around an Arduino Uno microcontroller, but ongoing research during the design of the robot revealed that several more attractive options existed (due to time constraints, however, the Uno platform was ultimately chosen as the controller for the robot controller). A Beaglebone Black embedded Linux development board was purchased to supercede the Arduino Uno for future experimentation.
